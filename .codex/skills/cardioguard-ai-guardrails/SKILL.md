# CardioGuard AI & Chatbot Guardrails Skill

## Scope
Use this skill when developing or modifying chatbot interfaces, AI diagnostic services, LLM integration modules, or chat history routers in:
- `backend`: AI integration services (`ai_service.py`), chat controllers, and prompt generators.
- `web_frontend` & `mobile_app`: Chat window components, AI telemetry suggestion popups, and chatbot hooks.

## Rules

### 1. Mandatory Medical Disclaimers
- **Standard Disclaimer**: Every conversational response generated by the AI chatbot to patients must explicitly include a medical disclaimer in Vietnamese, stating:
  `*Lưu ý: Đây là phân tích tự động từ AI và chỉ mang tính chất tham khảo, không thay thế cho chẩn đoán lâm sàng của bác sĩ chuyên khoa.*`
- **Emergency Warnings**: If the chatbot detects critical telemetry values or user prompts indicating an emergency (e.g., chest pain, shortness of breath), it must display a high-visibility warning to call local emergency services (115) immediately.

### 2. Error Boundary & Stack Trace Protection
- **No System Leakage**: Never return raw exception traces, internal database models, server hostnames, or API response errors to the user UI.
- **Graceful Fallbacks**: Wrap AI integration calls in robust `try/catch` (or `try/except`) blocks. If the LLM API fails, return a friendly generic message:
  `Xin lỗi, hệ thống AI đang bận hoặc gặp sự cố. Vui lòng thử lại sau.`

### 3. API Key & Request Safety
- **No Hardcoded Keys**: AI service keys (OpenAI, Gemini, HuggingFace) must strictly be loaded from secure environment variables.
- **Prompt Sanitization**: Sanitize user prompts to prevent injection attacks (e.g., attempting to bypass system guidelines to diagnose other diseases or dump system configs).
- **Request Limits**: Enforce maximum input token counts or character lengths on chat messages to prevent system abuse and high API cost consumption.
